JavaScript API overview

Buttons are native HTML elements styled by Scybud. I keep interactions vanilla and compatible with existing browser methods.

  • element.click() — trigger action programmatically
  • element.focus() — move keyboard focus
  • button.addEventListener('click', handler) — bind actions

Accessibility

  • Use native <button> semantics for keyboard and screen readers.
  • Confirm focus ring visibility and consistent contrast ratios.
  • Include clear action text for each button.

Accessibility guidance

I included this section to keep accessibility front and center across Scybud docs and components.

  • Prefer semantic HTML and native controls.
  • Keep all interactive elements reachable by keyboard.
  • Use color tokens that preserve contrast on dark backgrounds.
  • Document required ARIA roles only when native markup is insufficient.
